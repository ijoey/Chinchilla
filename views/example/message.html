This is an example on how to use Chinchilla, a RESTful framework in PHP

  1. Create a resource by creating a file in the resources folder. You can copy either example_resource.php or index_resource.php.

  2. Create a method in your new resource named the same as the HTTP method you want it to handle. For example, if you look at the example_resource.php file, you'll see a method called GET. This method responds to an HTTP GET request to a relative url /example/ (NOTE: you can specify the representation with file extensions like .html, .xml, .json, .js, or even .phtml for partial HTML). If you want to handle the HTTP POST method, then create a method called POST.

  3. Your methods will almost always look like this:

    				
	function GET(){
		/* logic here */
		$this->title = "An example using Chinchilla, A RESTful framework in PHP";
		$this->output = view::render("example/index", $this);
		return layout::render("default", $this);
	}

  4. The static view::render method will look in views/example/ for a file called index.html. The framework determines the file type (eg. html, xml, json, rss, atom, etc.) based on the url. If the url is /example.html, view::render will return the views/example/index.html file to the resource's $output property.

  5. layout::render will get layouts/default.html, which echo's the resources $output property.

  6. You'll also notice a themes folder in the file hierarchy. This folder contains themes that you might want to use for your site. There's a "default" theme in the code base now. The theme can be set in the theme_controller class. If the "views" folder hierarchy is defined in a theme folder within the themes folder, the framework will look there first to find any views. The view facility is how the framework implements the Representational part of REST. You can represent the data in any way by creating a view for it, according to the file type you specify. For instance, if I want a .doc representation, create view files appended with .doc. Check out the example resource and views to see what I mean.
